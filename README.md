# Hi, I'm Adarsh Agrahari 👋

Product leader. Author. GenAI faculty. Builder of small systems that explore large ideas.

I’m interested in intelligence — artificial and human.

For a long time, software was mostly about workflows:
forms, buttons, dashboards, APIs, permissions.

That world still matters.  
But something more interesting is happening now.

Software is becoming less like a tool, and more like a collaborator.  
Less about static interfaces, more about judgment.  
Less about where you click, more about what the system understands, remembers, and helps you do.

That shift is what I care about.

So I spend my time around the edges of:

- AI-native product design
- agents and orchestration
- memory for software systems
- workflow automation
- interaction layers for intelligent products
- product judgment in the age of AI

Most of my career has been in product leadership.  
But increasingly, I build to think.

## What I believe

The most important change in software is not that AI can generate text, images, or code.

It is that software can now participate in cognition.

That changes everything:
- how products should be designed
- what interfaces should optimize for
- what moats will look like
- how teams will work
- what “good judgment” in software even means

A lot of AI products today still feel like wrappers around a model.  
Useful, sometimes. Durable, rarely.

The more interesting opportunity is deeper:
to build systems that remember, adapt, coordinate, compress complexity, and increase a user’s ability to act.

That is the kind of software I want to understand.  
And build.
